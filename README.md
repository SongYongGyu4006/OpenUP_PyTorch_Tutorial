## 3. 신경망은 어떻게 학습되는가
기계학습은 경험 E를 통해 과제 T의 성능 P가 향상되는 과정으로 이해할 수 있다. (TEP)

  1. Task T : 해결하려는 문제
  2. Experience E : 데이터나 상호작용 경험
  3. Performance P : 정확도, 손실, F1 같은 평가 기준
  4. 모델보다 몬저 문제와 지표를 정의해야 한다.

## 3-1. 학습 설정의 종류
데이터가 어떤 형태로 주어지느냐에 따라 학습 방식과 목표가 달라진다.

  1. 지도학습 : 입력과 정답이 함께 주어짐
  2. 비지도학습 : 정답 없이 관계를 찾음
  3. 준지도학습 : 데이터 자체에서 학습 특성을 만듦
  4. 강화학습 : 행동과 보상을 통해 학습

## 3-2. 데이터는 분포에서 온다
우리가 가진 데이터셋은 현실 전체가 아니라 미지의 데이터 분포에서 관측된 일부 샘플

  1. 데이터셋은 현실 전체가 아니라 어떤 데이터 생성 분포에서 관측된 유한한 샘플
  2. 모델은 샘플을 외우는 것이 아니라 분포의 패턴을 배우려 한다.
  3. train set이 곧 현실 전체는 아니다.
  4. 데이터 편향은 모델 편향으로 이어질 수 있다.

## 3-3. Train/Validation/Test/Generalization
훈련 데이터에서 잘 맞는 것과 처음 보는 데이터에서 잘 맞는 것은 다르며 딥러닝의 핵심 목표는 높은 Generalization(일반화) 성능을 보이는 것

  1. Train : 파라미터를 실제로 업데이트하는 데이터
  2. Validation : 모델 선택과 하이퍼파리미터 점검
  3. Test : 마지막 한 번 성능을 확인하는 데이터
  4. Generalization : 보지 못한 데이터에서도 잘 작동하는 능력


## 3-4. Capacity, Underfitting, Overfitting
모델이 너무 단순하면 패턴을 못 배우고 너무 복잡하면 훈련 데이터를 지나치게 외울 수 있다.

  1. Capcity : 모델이 표현할 수 있는 함수의 복잡도(n차 함수)
  2. Underfitting : 구조가 너무 단순한 경우
  3. Overfitting : 훈련 데이터의 세부 노이즈까지 맞추는 경우
좋은 모델은 train 성능과 test 성능 사이 균형을 만든다


## 3-5. 1943 인간 뉴런에서 인공 뉴런으로
인공 뉴런은 생물학적 뉴런을 그대롤 복사한 것이 아니라 계산 가능한 추상 모델로 단순화한 것

  1. 입력 신호를 받는다
  2. 가중합을 계산한다.
  3. 비선형 함수(활성 함수)를 통과시킨다.
  4. 여러 뉴련을 연결하면 더 복잡한 함수를 만들 수 있다.

## 3-6. 1967 퍼셉트론과 선형 변환
퍼셉트론은 입력의 가중합을 계산하는 가장 기본적인 분류 단위 현대 딥러닝의 선형레이어으로 이어짐

  1. 입력 벡터와 가중치 벡터의 내적
  2. bias를 더해 기준점 이동
  3. activation을 통과시켜 출력

## 3-7. ML와 DNN
여러 층의 선형변환과 비선형 함수를 쌓으면 입력을 점점 더 유용한 표현으로 바꿀 수 있다. 

  1. MLP : 다층 퍼셉트론
  2. 입력층, 은닉층, 출력층으로 구성
  3. 각 층은 이전 층의 표현을 다시 변환한다
  4. 깊이가 더 깊어질수록 더 복잡한 패턴을 모델링할 수 있다.

## 3-8. Activation과 Output Transformation
활성함수가 있어야 깊은 층이 의미를 가지며 출력층은 문제 유형에 맞게 해석되어야 함

  1. ReLu : CNN/MLP 입문에서 가장 자주 만나는 활성함수
  2. Sigmoid : 이진 분류 확률 해석에 자주 사용
  3. Softmax : 다중 클래스 확률 분포로 변환


출력 해석은 문제 유형과 손실 함수에 연결된다.

## 3-9. 학습은 수치 최적화
신경망 학습은 손실함수를 최소화하는 파라미터를 찾는 수치 최적화 문제다.

  1. 모델은 파라미터를 가진 함수
  2. 예측과 정답의 차이를 loss로 수치화
  3. 최적화는 손실이 줄어드는 방향으로 파라미터를 수정
  4. "학습한다"는 말은 결국 예측을 정답의 분포에 가깝게 조정하는 것

## 3-10. Loss와 Accuracy
loss는 학습을 위한 미분 가능한 목표

accuracy는 사람이 이해하기 쉬운(목표로 하는) 성능 지표

  1. loss는 최적화가 직접 줄이는 대상
  2. 정확도는 맞춘 비율을 요약한 해석용 지표
  3. 분류에서는 crossEntropyLoss가 대표적

## 3-11. Gradient Descent와 optimizer
gradient는 손실을 줄이는 방향을 알려주고, optimizer는 그 정보를 이용해 파라미터를 실제로 갱신한다.

  1. gradient는 loss의 기울기
  2. learning rate는 한 번 이동하는 크기
  3. SGD, Adam은 대표적인 최적화
  4. 같은 모델이라도 최적화와 학습률에 따라 결과가 달라질 수 있음

## 3-12. 역전파와 autograd
역전파는 chain rule을 이용해 gradient를 뒤에서 앞으로 계산하는 원리

PyTorch의 autograd가 이를 자동 계산

  1. 순전파 : 예측 계산
  2. 역전파 : 반대로 미분

# 4. CNN
## 4-1. 1998 LnNet과 CNN의 시작
CNN은 이미지의 공간 구조를 활용해 문자인식과 이미지 인식 문제를 효과적으로 다루며 주목 받음

  1. 이미지는 2차원 구조를 가진 데이터
  2. 가까운 픽셀들은 함께 의미를 이룰 가능성이 높음
  3. CNN은 이 지역 패턴을 반복적으로 추출하다

## 4-2. 왜 이미지에는 CNN이 유리한가
CNN은 이미지의 지역성, 이동 불변성, 파라미터 공유를 구조적으로 반영하기 때문에 이미지 처리에 강점을 가짐

  1. 커널을 사용하여 여러 위치에 반복적용

## 4-3. Convolution, Filter, Feature Map
컨볼루션연산은 작은 필터를 이미지 전역에 적용해 특정 패턴의 존재를 특징맵으로 바꿔 주는 연산

## 4-4. Stride, Padding, Pooling

  1. Stride : 필터가 이동하는 간격
  2. Padding : 빈공간 메꾸기

## 4-5. CNN 아키텍쳐 : Feature Extractor와 Classifier
CNN은 앞쪽에서 특징을 추출하고 뒤쪽에서 그 특징을 바탕으로 분류를 수행하는 구조

  1. 앞단 : 컨볼루션-활성-풀링층 반복
  2. 중간 : 점점 추상적인 시각 특징 학습
  3. 뒷단 : 평탄화 또는 전역 풀링 후 분류층 연결
  4. 분류기는 최종 logits를 만든다.

## 4-6. AlexNet과 ImageNet
AlexNet은 대규모 데이터, GPU, 깊은 CNN의 결합이 실제 이미지 인식 성능을 크게 끌어올릴 수 있음을 보여주었다.

  1. ImageNet은 대규모 이미지 분류 벤치마크였다.
  2. AlexNet은 깊은 CNN 구조와 ReLU를 사용했다.
  3. dropout과 GPU 학습이 실용적 성능 향상에 기여했다.

## 4-7. ResNet
ResNet은 shortcut connection을 통해 매우 깊은 네트워크도 더 안정적으로 학습할 수 있게 만들었다.

  1. 깊은 네트워크는 최적화가 어려워 질 수 있다.
  2. ResNet block은 입력을 뒤로 전달하는 지름길을 만든다.

## VIT
ViT는 픽셀 전체를 한 번에 보는 대신 patch를 token처럼 다루며 그 관게를 attention으로 학습

  3. attention은 멀리 떨어진 patch 관계도 직접 본다
  4. CNN은 local bias와 다른 방식으로 구조를 학습
